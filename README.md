# 🔧 Анализ технического состояния МГП методами машинного обучения

Комплексный проект по анализу данных внутритрубной диагностики (ВТД) магистрального газопровода за период 2015–2024 гг. Включает разведочный анализ, выявление пространственных горячих зон, валидацию классификации опасности и прогнозирование роста дефектов с помощью 5 ML-моделей.

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
git clone https://github.com/YOUR_USERNAME/pipeline-ml.git
cd pipeline-ml
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Подготовка данных

Положите CSV-файлы ВТД в `data/raw/`:
```
data/raw/
├── vtd_2015.csv
├── vtd_2020.csv
├── vtd_2022.csv
└── vtd_2024.csv
```

### 3. Запуск полного пайплайна

```bash
# Шаг 1: Обработка данных и матчинг
python src/data/loader.py
python src/data/matching.py

# Шаг 2: EDA
python src/eda/statistics.py
python src/eda/hotspot.py
python src/eda/danger.py

# Шаг 3: Обучение всех моделей
python src/models/train_all.py

# Или запустить всё одной командой:
python run_pipeline.py
```

### 4. Jupyter notebooks (рекомендуется для изучения)

```bash
jupyter lab
# Открыть notebooks/ в порядке 01 → 04
```

### 5. Запуск веб-приложения

```bash
# Streamlit MVP
streamlit run app/main.py

# FastAPI (если выбрана эта версия)
uvicorn app.main:app --reload
```

## 📋 Требования к данным

Каждый CSV-файл ВТД должен содержать следующие обязательные столбцы:

| Столбец | Описание | Тип |
|---------|----------|-----|
| `dist_ref` / `dist_m` | Расстояние от начала (мм или м) | float |
| `pipe_num` | Номер трубы | str/int |
| `depth_pct` | Глубина дефекта, % от толщины стенки | float |
| `len_mm` | Длина дефекта, мм | float |
| `wid_mm` | Ширина дефекта, мм | float |
| `dist_lw` | Расстояние до продольного шва, мм | float |
| `dist_cw` | Расстояние до кольцевого шва, мм | float |
| `tube_len` | Длина трубы, мм | float |
| `kbd` | КБД (коэффициент безопасного давления) | float |
| `pf` | Давление разрушения, МПа | float |
| `danger` | Категория: (a), (b), (c) | str |

> Точные названия столбцов настраиваются в `src/utils/config.py`

## 🧠 Методология ML

### Задача
**Регрессия**: по истории измерений 2015–2022 предсказать `depth_pct_2024`.

### Ключевое решение: раздельная подготовка данных

| Модель | Формат данных | Скейлинг | N объектов |
|--------|--------------|---------|-----------|
| GradientBoosting | Wide: [depth_15,20,22 + Δ + геом.] | Нет | 2066 |
| RandomForest | Wide + depth_ratio_20_22 | Нет | 2066 |
| RNN | Seq: (N, 3_шага, 6_признаков) | MinMaxScaler [-1,1] | 1363 |
| MLP | Wide + 22 признака | RobustScaler | 2066 |
| IsoForest | 4 года × 5 = 20 признаков | Нет | 2093 |

### Почему RNN лучше

RNN получает данные как **реальный временно́й ряд**: скрытое состояние накапливает тренд роста через шаги 2015→2020→2022. GB/RF видят набор признаков, не динамику.

## 📈 Горячая зона 136–145 км

Зона с **3.7× превышением** плотности дефектов на протяжении всех 9 лет наблюдений. Дефекты в зоне: длиннее (+62%), шире (+76%), ближе к продольному шву (-29%).

Гипотезы: изгиб трубопровода, агрессивный грунт, повреждение изоляции.
